# Workflow

이 스킬은 실무 작업을 일관된 순서로 진행하기 위한 실행 라우터입니다.

기존 표준 문서를 대체하지 않습니다. 코드 스타일, 함수 네이밍, TanStack Query, 테스트 설계가 필요하면 이 플러그인의 기존 standards/skills를 함께 적용합니다.

## Modes

### scan

수정 없이 프로젝트와 요구사항을 파악합니다.

목표:

1. 프로젝트 구조와 실행 방식을 파악합니다.
2. 요구사항을 성공 기준으로 재정의합니다.
3. 기존 구현 패턴과 관련 파일을 찾습니다.
4. 가장 작은 안전한 구현 계획과 검증 계획을 제안합니다.

규칙:

- 파일을 수정하지 않습니다.
- 추측하지 말고 파일, scripts, 타입, 기존 패턴을 근거로 말합니다.
- 새 의존성 추가를 기본 선택지로 제안하지 않습니다.
- 과한 리팩터링, 전역 구조 변경, 스타일 재작성은 계획에서 제외합니다.
- 요구사항이 materially branching하면 구현 전에 짧게 질문합니다.

출력:

```text
요구사항 요약:
- ...

성공 기준:
- ...

관련 파일:
- path: 이유

기존 패턴:
- ...

구현 계획:
1. ...
2. ...
3. ...

검증 계획:
- ...

리스크:
- ...
```

### do

성공 기준을 만족하는 가장 작은 변경을 구현합니다.

규칙:

- 기존 패턴을 우선합니다.
- 새 의존성은 사용자가 명시적으로 요청하지 않는 한 추가하지 않습니다.
- 요구사항과 무관한 리팩터링을 하지 않습니다.
- 전역 상태, 공용 API, shared component를 바꿀 때는 영향 범위를 먼저 확인합니다.
- 생성된 코드를 그대로 믿지 말고 diff를 읽으며 진행합니다.
- 동작 변경이 있으면 가능한 가장 가까운 테스트를 추가하거나 갱신합니다.
- 시간이 제한된 상황에서는 완벽한 추상화보다 요구사항 충족과 회귀 방지를 우선합니다.

출력:

```text
구현 내용:
- ...

변경 파일:
- ...

검증:
- 실행한 명령 또는 확인한 동작

남은 확인 사항:
- ...
```

### fix

실패를 추측으로 고치지 않고, 증거 기반으로 원인을 좁혀 수정합니다.

절차:

1. 실패 명령과 에러 메시지를 확인합니다.
2. stack trace 또는 실패 위치에서 관련 파일을 엽니다.
3. 기대 동작과 실제 동작의 차이를 한 문장으로 정리합니다.
4. 원인 후보를 최대 2개로 줄입니다.
5. 코드 근거가 있는 원인만 수정합니다.
6. 실패했던 명령을 다시 실행합니다.

규칙:

- 로그 없이 원인을 단정하지 않습니다.
- 관련 없는 리팩터링을 하지 않습니다.
- 실패를 숨기기 위해 테스트를 약화하지 않습니다.
- 타입 단언, 무시 주석, catch 후 무시 같은 회피성 수정은 마지막 수단입니다.

출력:

```text
실패 증거:
- ...

원인:
- ...

수정:
- ...

재검증:
- ...

남은 리스크:
- ...
```

### check

작업을 제출하거나 공유하기 전에 현재 상태를 검증하고 요약합니다.

확인할 것:

- `git status`
- `git diff`
- package scripts
- 타입체크, lint, test, build 중 가능한 검증
- UI 작업이면 브라우저 또는 수동 확인이 필요한 흐름

규칙:

- 검증하지 않은 것을 통과했다고 말하지 않습니다.
- 실패한 검증이 있으면 먼저 원인을 해결합니다.
- 시간이 없어 못 한 검증은 `미검증`으로 명시합니다.
- 변경 요약은 사용자가 이해할 수 있는 동작 중심으로 작성합니다.

출력:

```text
변경 요약:
- ...

성공 기준 점검:
- [x] ...
- [ ] ...

검증 결과:
- ...

변경 파일:
- ...

남은 리스크 / 미검증:
- ...
```

## Default Engineering Rules

- Existing patterns first.
- No new dependencies unless explicitly requested.
- Small, reversible diffs.
- Evidence before claims.
- Verify before saying done.
- State unverified gaps clearly.
